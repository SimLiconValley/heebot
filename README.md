# 🤖 히봇 (heebot)

Go 백엔드와 Python(`yt-dlp`) 엔진을 결합하여 빠르고 안정적인 멀티미디어 서빙을 지원하는 디스코드 봇입니다. 

## 설치 방법

**FFmpeg**이 설치되어 있어야 합니다.
시스템 환경 변수에 **FFmpeg** 등록 필수

`go build -o heebot`

`chmod +x heebot`

이후 `heebot` 을 실행해주세요.

## ✨ 주요 기능

### 0. help
`/help` : 현재 보고 계시는 명령어 도움말 창을 화면에 띄워줍니다.

### 1. 💬 기본 상호작용 (텍스트 반응)

`안녕` 이라고 쳐보세요. `그래 잘 살고 있니?`라고 즉시 대답합니다.



### 2. 🎥 유튜브 다운로드
`/yt url:<유튜브주소> [quality:<화질/포맷>] [shared:<공개여부>]`

- 단일 유튜브 영상을 고속으로 분석하고 안전하게 다운로드할 수 있는 웹 링크를 생성합니다.
- `quality (선택)`: `1080p`, `720p (기본값)`, `480p` 화질 지정 혹은 `MP3 (최고음질)` 추출 포맷을 선택할 수 있습니다.
- `shared (선택)`: 공유 설정 시 서버원 모두가 볼 수 있으며, 선택하지 않거나 비공개로 두면 나에게만 결과가 보입니다.
- ⚠️ 유튜브 재생목록(Playlist) 주소는 이 명령어로 직접 다운로드할 수 없습니다.

### 3. 🔍 카톡메시지로부터 유튜브 URL 추출

`/mty content:<긴 텍스트> [print_at:<출력 위치>] [count:<출력 개수>]`
- 카톡 공지사항이나 장문의 글 속에서 다른 웹 링크는 거르고 오직 유튜브 주소만 정확히 골라냅니다.
- 만약 주소 중에 유튜브 재생목록(Playlist) 주소가 포함되어 있다면, 내부 시스템(yt-dlp)이 백그라운드에서 작동하여 안에 숨겨진 모든 개별 영상의 제목과 URL을 전부 파싱하여 일괄 전개해 줍니다.
- 10개 단위 자동 분할 출력: 디스코드 글자 수 제한을 초과하지 않도록 10개씩 깔끔하게 페이지를 나누어 가독성 높은 리스트로 출력합니다.
- 유연한 출력 위치 (print_at): 결과를 채널에 '전체 공개', 나만 볼 수 있는 '나만 보기(Ephemeral)', 혹은 깔끔하게 갠톡으로 받는 'DM으로' 중 선택하여 전송받을 수 있습니다. (기본값: 나만 보기)
- 최대 개수 제한 (count): 출력할 URL의 최대 개수를 직접 지정할 수 있으며, 0을 입력하면 제한 없이 모든 링크를 전개합니다. (기본값: 0)
- 전개된 주소는 `/yt` 명령어로 개별 다운로드할 때 매우 유용하게 쓰일 수 있습니다.

## 🔨 구현
1. 서버 내부 저장소에 yt-downloader.py로 영상을 다운로드 후 이를 파일 서버로 이용자에게 제공
2. 파일마다 유효기간을 설정하여 시간이 지나면 자동 삭제.
